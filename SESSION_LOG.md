# Blitz — Session Log

Built in one session as a native macOS menu-bar STT app to replace Wispr Flow. This log captures what was built, non-obvious decisions, and how to pick things up next time.

## Status

- Builds cleanly on Xcode 26.4 / macOS 26.4 / Swift 5.10, target macOS 14+.
- Runs, records, transcribes, pastes. History + retry shipped.
- Ad-hoc signed (no Team ID). Non-sandboxed.

## Runtime flow

```
User presses binding → CGEventTap (HotkeyManager) → AppCoordinator.hotkeyPressed
  → AudioCapture.start() (AVCaptureSession, pinned to built-in mic)
  → RecordingOverlay.show()

User releases binding (hold-to-talk)     → hotkeyReleased
User taps Right-Command during hold      → latched, keeps recording
User presses binding again (latched)     → hotkeyReleased

hotkeyReleased → AppCoordinator.finishRecording
  → AudioCapture.stop() returns 16 kHz mono PCM16 WAV Data
  → TranscriptHistory.appendPending(mode, audioData)
  → OpenAIClient.transcribe(wav)
  → if mode.needsPostProcessing: OpenAIClient.rewrite(text, systemPrompt)
  → TranscriptHistory.markSucceeded(id, text, raw)       ← on success
    TranscriptHistory.markFailed(id, error)              ← on failure
  → TextInjector.insert(text)                            ← on success only
  → presentError(...) with "saved in History, can retry" ← on failure only
```

## Hotkeys

Defaults, all remappable in Settings → Hotkeys:

| Mode    | Binding              | Post-processing |
|---------|----------------------|-----------------|
| Exact   | Right-Option         | none            |
| Written | Ctrl+Opt+1           | gpt-4o-mini     |
| Rage    | Ctrl+Opt+2           | gpt-4o-mini     |
| Emoji   | Ctrl+Opt+3           | gpt-4o-mini     |

**Latch:** while a modifier-only binding is held, tap `Right-Command` (left of Right-Option) → latched. Release primary modifier, keep recording. Press primary modifier once more to stop. Hardcoded in `HotkeyManager.latchTrigger` — swap to e.g. `.rightShift` if preferred.

**Why not Space:** Space conflicts with Raycast / Spotlight / iTerm toggles depending on user setup.
**Why not Fn:** macOS Dictation claims Fn, and CGEventTap delivery for Fn-combos is inconsistent across keyboards.

## Non-obvious decisions

### AVCaptureSession, not AVAudioEngine
AVAudioEngine with HAL-level device override (`AudioUnitSetProperty(kAudioOutputUnitProperty_CurrentDevice)`) broke audio capture on macOS 26 — tap never fired, silent failure. Switching to AVCaptureSession with `AVCaptureDevice(uniqueID:)` matched against the built-in mic's CoreAudio UID is:
- Reliable (no AVAudioEngine internal state).
- Avoids A2DP→HFP profile switch on Bluetooth (YouTube etc. keep playing cleanly).
- Uses explicit `AVCaptureAudioDataOutput` → CMSampleBuffer → AVAudioConverter → 16 kHz mono Int16 → WAV.

### Built-in mic detection
Three-tier fallback in `AudioCapture.selectInputDevice()`:
1. CoreAudio HAL: enumerate devices, find one with `kAudioDeviceTransportTypeBuiltIn` + input streams. Take its UID, look up via `AVCaptureDevice(uniqueID:)`.
2. Name heuristic: mic whose `localizedName` contains "built-in" / "macbook" / "internal" / "imac" / "mac mini" / "mac studio".
3. `AVCaptureDevice.default(for: .audio)`.

### HotkeyManager state machine
```
enum ActiveState { case none, holding(Mode), latched(Mode) }
```
`bindingPressed` and `bindingReleased` are the only state transitions. Space-latch was replaced with modifier-latch (`Right-Command`) because Space is a global shortcut magnet.

### Clipboard restore in TextInjector
Snapshots all pasteboard items (types + data), writes the transcribed text, synthesizes Cmd+V, sleeps 180ms for the receiving app to read, then restores. On retry-from-history, text goes on the pasteboard and stays there (user will paste manually — cursor is usually not where they want it anymore).

### History storage
- `~/Library/Application Support/Blitz/history.json` — metadata, JSON with ISO-seconds timestamps.
- `~/Library/Application Support/Blitz/audio/<uuid>.wav` — raw PCM16 WAV, only kept for pending/failed entries. Deleted on `markSucceeded`.
- Cap: 100 entries; oldest dropped first (including their audio files).

## Build / deploy commands

Always prefix with `DEVELOPER_DIR` since `xcode-select` was never switched via sudo:

```bash
# One-time after adding/removing/renaming Swift files
cd ~/Desktop/Blitz && xcodegen generate

# Build
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
  xcodebuild -project Blitz.xcodeproj -scheme Blitz \
  -configuration Debug -derivedDataPath build build

# Deploy — MUST use ditto or rm+cp (not plain cp -R on existing .app!)
pkill -x Blitz 2>/dev/null; sleep 1
rm -rf ~/Desktop/Blitz/Blitz.app
ditto ~/Desktop/Blitz/build/Build/Products/Debug/Blitz.app ~/Desktop/Blitz/Blitz.app
open ~/Desktop/Blitz/Blitz.app
```

**Why ditto:** `cp -R src dst` where `dst` is an existing directory copies `src` INTO `dst` (merges), leaving stale binaries. Cost me ~20 min of debugging a stale-binary crash that referenced source lines already deleted. UUID matched the new build but mtime on the deployed executable was from the previous build. Use `ditto` or `rm -rf dst && cp -R src dst`.

## Permissions

- **Mic:** `NSMicrophoneUsageDescription` in Info.plist, prompted lazily on first `AVCaptureSession.startRunning()`.
- **Accessibility:** required for CGEventTap + CGEvent-based Cmd+V. Prompted at launch via `AXIsProcessTrustedWithOptions`. After each rebuild the ad-hoc cdhash changes, so the TCC entry may invalidate — user re-toggles the Accessibility checkbox and relaunches.
- **No App Sandbox.** Sandboxed apps can't post events to arbitrary foreground apps.

## Known limitations

- **No selectable input device UI.** Built-in mic only (intentionally, to prevent Bluetooth profile switching). A Settings dropdown is a future nice-to-have — the plumbing is in `selectInputDevice()` and a user-override could bypass the built-in preference.
- **Short recordings (<0.5s)** often produce empty Whisper responses. Not retried automatically; shows up as failed entry in History.
- **Accessibility permission invalidated on rebuild** for ad-hoc signed apps. Paid developer ID would solve this — not currently configured.
- **Latch trigger hardcoded** to Right-Command. Swap in `HotkeyManager.swift`: `var latchTrigger: ModifierKey = .rightCommand` → change to another `ModifierKey` case.
